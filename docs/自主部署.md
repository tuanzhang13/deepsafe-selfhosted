# 自主部署版 DeepSafe

自主部署版在客户自有服务器运行，**完全离线验证**，不依赖 DeepSafe 云端服务端。分为 **免费版** 与 **订阅版** 两档。

| 能力 | 免费版 | 订阅版 |
|------|--------|--------|
| 管理端 | ✓ | ✓ |
| 服务端（平台管理） | ✗ | ✗ |
| 应用数 | 3 | 100 |
| 单批卡密 | 50 | 1000 |
| 多开设备 | 1 | 5 |
| 代理分销 | ✗ | ✓ |
| 注册账号 | ✓ | ✓ |

## 快速启动（Docker · Ubuntu）

### 一句话部署

```bash
curl -fsSL https://raw.githubusercontent.com/tuanzhang13/deepsafe-selfhosted/main/deploy/install.sh | bash
```

Windows PowerShell：

```powershell
irm https://raw.githubusercontent.com/tuanzhang13/deepsafe-selfhosted/main/deploy/install.ps1 | iex
```

**推荐环境：** Ubuntu 22.04 / 24.04 LTS，已安装 [Docker Engine](https://docs.docker.com/engine/install/ubuntu/) 与 Docker Compose 插件。

### 手动启动

或手动：

```bash
cd deploy
cp .env.selfhosted.example .env.selfhosted
# 编辑 .env.selfhosted：JWT_SECRET

docker compose -f docker-compose.selfhosted.yml pull
docker compose -f docker-compose.selfhosted.yml up -d
```

Windows 开发机同样可用 PowerShell：

```powershell
cd deploy
powershell -File start-selfhosted.ps1
```

启动后访问：

| 地址 | 说明 |
|------|------|
| http://127.0.0.1:8088 | **管理端**（默认端口，可在 `.env.selfhosted` 改 `WEB_PORT`） |
| http://127.0.0.1:8088/api/edition/info | 实例状态 / 机器码 |

默认账号：`demo / author123`（首次启动自动创建，也可注册新账号）。

**容器组成（均为 Ubuntu/Debian 系 Linux 镜像）：**

| 服务 | 基础镜像 | 说明 |
|------|----------|------|
| `api` | Ubuntu 24.04 | Go 后端（`EDITION=selfhosted`） |
| `author` | Ubuntu 24.04 + Nginx | 管理端静态页面 |
| `web` | Ubuntu 24.04 + Nginx | 统一网关（8088 端口） |
| `mysql` | mysql:8.0 | 数据存储 |
| `redis` | redis:7-bookworm | 缓存 |

停止：`docker compose -f docker-compose.selfhosted.yml down`

查看日志：`docker compose -f docker-compose.selfhosted.yml logs -f api`

查询机器码（Linux）：

```bash
chmod +x ../scripts/machine-id.sh
../scripts/machine-id.sh http://127.0.0.1:8088
```

## 本地开发（无 Docker）

```powershell
cd backend
$env:MYSQL_USER="root"
$env:MYSQL_PASSWORD="root"
$env:MYSQL_DATABASE="deepsafe"
$env:REDIS_MEMORY="true"
$env:EDITION="selfhosted"
$env:LICENSE_PUBLIC_KEY="<64位hex公钥>"
go run ./cmd/server
```

```powershell
cd web\author
npm run dev
```

## 生成授权文件

1. 启动服务后，在管理端「订阅与套餐」页复制 **机器码**，或请求：

   ```
   GET /api/edition/info
   ```

   响应中的 `machine_id` 即为机器码。

2. 使用 `license-gen`（仅 DeepSafe 内部使用，不随部署包分发）：

   ```powershell
   go build -o bin/license-gen.exe ./tools/license-gen

   # 订阅版（绑定机器码，365 天有效）
   .\bin\license-gen.exe --plan full --machine <机器码> --days 365 --output license.lic

   # 免费版试用授权（可选，一般无需文件即为免费版）
   .\bin\license-gen.exe --plan free --machine <机器码> --output license-free.lic
   ```

3. 首次运行 `license-gen` 会生成 `license-ed25519.key` 与 `.pub` 公钥文件。将公钥 hex 写入部署环境变量 `LICENSE_PUBLIC_KEY`。

4. 客户将 `license.lic` 在管理端「订阅与套餐」页上传，或复制到 `LICENSE_PATH`（默认 `./license.lic`）后重启服务。

## 环境变量

| 变量 | 说明 |
|------|------|
| `EDITION` / `NETVERIFY_EDITION` | 设为 `selfhosted` |
| `LICENSE_PATH` | 授权文件路径，默认 `./license.lic` |
| `LICENSE_PUBLIC_KEY` | Ed25519 公钥 hex（验证签名必需） |
| `MACHINE_CODE` | 可选，覆盖自动计算的机器码（测试用） |

## 与云端版的区别

- **配额按实例**：同一部署内所有作者共享实例级配额（免费/订阅），而非 SaaS 的按作者套餐。
- **无服务端**：`/api/admin/*` 不可用；授权卡在自主部署版中不能用于升级。
- **离线授权**：订阅版通过 `.lic` 文件激活，不联网回传。

## 机器码脚本

```powershell
powershell -File scripts\machine-id.ps1 -ApiBase http://localhost:8080
```

若 API 未启动，脚本会提示手动设置 `MACHINE_CODE` 环境变量。
